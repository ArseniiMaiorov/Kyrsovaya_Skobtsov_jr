# Спецификация автокодировщика

## Роль в проекте
- Автокодировщик обязателен по постановке.
- Для multiclass-ветки основной режим: предобучение представлений на train-части без использования целевых меток в целевой функции AE.

## Использование данных
- Источник для AE: train-часть набора (возможна неразмеченная подвыборка, если будет предоставлена).
- Валидация AE ведется отдельно от финальной test-оценки классификатора.

## Обязательная проверка
- Тест на синтетике: аномальные примеры должны иметь большую `reconstruction error`, чем нормальные (в среднем или по квантили).
