# Контракт данных

## Источник данных
- Датасет предоставлен кафедрой или преподавателем.
- Размеченный файл: `data/MKA_TMI_labels.xls`.
- Неразмеченный файл: `data/MKA_04.2015_unlabeled.csv`.

## Форматы
- Размеченный: `XLS`, лист `Лист1`.
- Неразмеченный: `CSV`, кодировка `utf-8`, разделитель `,`.

## Схема
- Целевая колонка в размеченном датасете: `Class`.
- Допустимые метки: `{0, 1, 2}`.
- Признаковые колонки: 49 общих колонок ТМИ.
- В неразмеченном датасете колонка `Class` отсутствует.

## Фактические размеры (на текущем наборе)
- Размеченный: `2679 x 50`.
- Неразмеченный: `15243 x 49`.
- Распределение классов в размеченном:
  - `0`: 2356
  - `1`: 218
  - `2`: 105

## Пропуски
Считаются пропусками:
- `""`
- `NA`, `N/A`, `NaN`, `nan`, `null`, `None`
- `-999`, `-9999`

## Временная ось
- Если есть явная колонка времени (`time/timestamp/...`) — используется сортировка по ней внутри серии.
- Если явной колонки времени нет — временем считается порядок строк в файле.
- Если есть колонка серии (`session/pass/orbit/track/series_id`) — окна строятся только внутри серии.
- Если такой колонки нет — весь файл считается одной серией.

## Поведение валидации
`src/data/io.py` и `src/data/validation.py` должны останавливать запуск с понятной ошибкой, если:
- отсутствует `Class` в размеченном наборе;
- метки вне `{0,1,2}`;
- `Class` присутствует в неразмеченном наборе;
- признаки размеченного и неразмеченного наборов не совпадают;
- число признаков отличается от 49;
- встречаются нечисловые признаки после парсинга null-токенов;
- после удаления полностью пустых строк датасет пуст.
