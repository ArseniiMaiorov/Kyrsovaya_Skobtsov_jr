# Контракт данных

## Источник данных
- Датасет предоставляется кафедрой или преподавателем.
- Рабочее расположение в проекте: `data/tmi.csv`.

## Формат файла
- Формат: `CSV`
- Кодировка: `utf-8`
- Разделитель: `,`

## Схема
- Целевая колонка: `Class`
- Допустимые метки: `{0, 1, 2}`
- Признаковые колонки: все колонки, кроме `Class`.

## Пропуски
Считаются пропусками:
- `""`
- `NA`, `N/A`, `NaN`, `nan`, `null`, `None`
- `-999`, `-9999`

## Временная ось
- Для режима `time_windows` данные должны иметь упорядоченную последовательность измерений внутри непрерывного ряда или сеанса.
- Конкретное поле времени фиксируется после получения фактического датасета и дописывается в этот документ.

## Поведение валидации
`src/data/validation.py` обязан прерывать запуск с понятной ошибкой, если:
- отсутствует `Class`;
- найдены метки вне `{0,1,2}`;
- отсутствуют обязательные признаки (если они заданы контрактом).
