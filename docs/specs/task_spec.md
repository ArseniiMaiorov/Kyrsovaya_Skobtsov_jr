# Спецификация задачи

## Выбранная постановка
- Вариант: `1.1`
- Тип: `multiclass`
- Смысл меток:
  - `0`: штатное состояние
  - `1`: отказ
  - `2`: сбой

## Правила
- В текущем цикле разработки используется только multiclass-постановка.
- Любая бинаризация допускается только как отдельный эксперимент и не заменяет основной контракт.

## Валидация
`src/data/validation.py` проверяет, что после загрузки метки принадлежат только множеству `{0,1,2}`.
